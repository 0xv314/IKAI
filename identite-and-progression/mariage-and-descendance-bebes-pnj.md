# Mariage & Descendance (Bébés PNJ)

Le système de **Mariage & Descendance** permet à deux joueurs de former une union et de concevoir un **enfant PNJ**. Le mariage ne dépend **ni du sexe**, ni d’un rôle narratif : il s’agit d’un **pacte magique** entre deux joueurs. L’enfant est **lié au compte** de ses parents, **non transférable** et **non vendable**.

Ce système renforce la **dimension sociale**, la **coopération**, et l’**ancrage long terme** dans l’univers.

***

### Le Mariage

Le mariage nécessite la création d’une **Alliance Magique**. Cette alliance ne peut être obtenue qu’en **Forge**, via une recette spéciale.

#### Alliance Magique (Condition Obligatoire)

| Élément          | Détail                                                    |
| ---------------- | --------------------------------------------------------- |
| Obtention        | **Forgeron uniquement** (recette spéciale)                |
| Ressources       | Alliance magique + Pierre magique + Matériaux spécifiques |
| Taux de réussite | **100%**                                                  |
| Difficulté       | **Réunir les ressources**, pas le craft                   |

Sans **Alliance Magique**, **aucun mariage** et **aucune descendance** n’est possible. Même la richesse ne permet pas de l’acheter directement → elle doit **être produite**.

***

### Rupture (Divorce)

Le divorce est possible **à tout moment**, en déposant **les 2 bagues** dans un onglet dédié.

| Condition           | Effet                                                |
| ------------------- | ---------------------------------------------------- |
| Paiement en **SOL** | Divorce instantané                                   |
| Coût                | Augmente selon le **nombre d’enfants** conçus        |
| Remariage           | Coût croissant selon le **nombre d’enfants cumulés** |

Ce système **dissuade la rotation stratégique** et encourage les unions stables.

***

### Descendance (Bébés PNJ)

Une fois mariés, les joueurs peuvent lancer une **Incubation** de 7 à 9 semaines. L’enfant naît directement **à 8–10 ans**, donc immédiatement **fonctionnel**.

| Propriété           | Valeur                           |
| ------------------- | -------------------------------- |
| **Statut**          | PNJ interne, non jouable         |
| **Vente / Échange** | **Impossible**                   |
| **Transfert**       | **Impossible**                   |
| **Utilité**         | Collecte passive de ressources   |
| **Risque**          | Mort définitive si non entretenu |

La descendance empêche toute **spéculation** et toute **production industrielle d’unités**, contrairement à Axie, Mir4 ou Genopets.

***

### Héritage de Classe & Rang

L’enfant hérite **automatiquement** d’une base génétique liée aux parents :

#### Héritage de Classe

| Cas                                                  | Probabilité |
| ---------------------------------------------------- | ----------: |
| Hérite des **deux classes** des parents              |     **60%** |
| Hérite d’une seule classe (aléatoire entre les deux) |     **40%** |

La Seconde Classe est toujours **moins puissante**, pour préserver l’équilibre.

#### Héritage de Rang

| Cas                                                     | Probabilité |
| ------------------------------------------------------- | ----------: |
| Hérite **à 60% du rang le plus élevé** des deux parents |     **60%** |
| Hérite **à 40% du rang le plus bas** des deux           |     **40%** |

Ce système **récompense les joueurs expérimentés** sans créer d’effet boule de neige.

***

### Rôle de l’Enfant

L’enfant permet une **collecte passive** :

| Facteur                                  | Impact sur récolte                                     |
| ---------------------------------------- | ------------------------------------------------------ |
| Rang de l’enfant                         | Détermine la quantité                                  |
| Classe de l’enfant                       | Détermine le type de ressource                         |
| Entretien (nourriture / soin / présence) | Influence l’efficacité                                 |
| Négligence totale                        | **Mort définitive → Pierre Tombale** dans l’inventaire |

Le joueur choisit **combien d’enfants il souhaite gérer**.

Aucune limite **non naturelle** → Mais plus d’enfants = **plus d’entretien** = **plus de responsabilité**.

***

### Pourquoi ce système est durable

| Problème historique (Axie, Mir4, Genopets) | Effet                     | Solution IKAI                                |
| ------------------------------------------ | ------------------------- | -------------------------------------------- |
| Reproduction illimitée                     | Effondrement des prix     | Enfant **non échangeable**                   |
| Inflation sur le marché                    | Perte de valeur           | Production **hors marché**, usage seulement  |
| Farms industrielles                        | Destruction de l’économie | Entretien obligatoire + Mort définitive      |
| Métagame basé sur profit                   | Perte d’intérêt du jeu    | Descendance = **outil**, pas actif financier |

***

### Résumé

* Le mariage n’est **pas lié au sexe**.
* Il nécessite une **Alliance Magique**, produite par **la Forge**.
* Le divorce existe, mais **à coût croissant** selon la descendance.
* L’enfant est **PNJ non spéculatif**, lié définitivement au compte.
* La descendance permet une **collecte passive**, avec **risque réel** de **mort**.
* L’héritage de Classe et de Rang est **probabilisé**, contrôlé, et équilibré.
